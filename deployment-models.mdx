## The One Invariant

**Your data plane never leaves your infrastructure.** Agent pods, evidence Postgres, SPIRE Server, OPA, Vault — all run in your cluster. The Hexr Control Plane handles licensing, cluster registration, and fleet health. It never sees agent evidence, credentials, or PII.

The only Hexr-operated endpoint your cluster calls: `api.hexr.cloud/v1/heartbeat` — 5 signed scalars every 60 seconds (`cluster_id`, `agent_count`, `evidence_row_count_24h`, `license_id`, `version`). That is the complete network surface.

## Route B — Self-Hosted Kubernetes

All public deployments use Route B. Your cluster. Your Postgres. Your trust root.

Works on: **AWS EKS · GCP GKE Standard · Azure AKS · on-premises K8s**

```bash
helm install hexr-runtime hexr/hexr-runtime \
  -n hexr-system \
  -f values.yaml
```

## PKI Mode — How Your Cluster Gets Its Trust Root

The difference between modes is who signs the intermediate CA certificate inside your cluster. Agent code, Helm chart, and SDK are identical across all modes.

<CardGroup cols={2}>
  <Card title="Mode A — Hexr-Managed PKI" icon="fingerprint">
    Hexr Control Plane acts as SPIRE `UpstreamAuthority`. Your cluster's SPIRE Server is issued an intermediate CA certificate signed by Hexr's root. Fastest path to production. No PKI infrastructure required on your side.

    **Suited for:** Teams without an existing PKI. Connected deployment with `api.hexr.cloud` reachable.
  </Card>
  <Card title="Mode B — BYO Vault PKI" icon="lock">
    Your HashiCorp Vault PKI backend is the intermediate CA. Hexr CP acts as the upstream authority above Vault. Your AKS/EKS cluster runs a nested SPIRE Server. Per-process SVIDs are minted locally — evidence stays in your VPC.

    **Suited for:** HIPAA BAA scope, FedRAMP IL5, any deployment where your security team controls the signing root. Required for Ring 2 (healthcare) and Ring 3 (defense/govtech) buyers.
  </Card>
</CardGroup>

<Card title="Mode C — Fully Air-Gapped" icon="shield-halved" href="/self-hosted/air-gapped">
  Zero outbound connectivity. No `api.hexr.cloud` egress. All container images pre-loaded from your internal registry. Customer-owned root CA, no Hexr CP involvement in PKI. License validated offline via JWT. Required for FedRAMP IL5, ITAR, CMMC L3, or environments with network egress controls.
</Card>

## Comparison

| | Mode A | Mode B | Mode C (Air-Gapped) |
|---|---|---|---|
| **Infrastructure** | Your K8s cluster | Your K8s cluster + HashiCorp Vault | Your K8s cluster, isolated network |
| **PKI root** | Hexr CP intermediate CA | Your Vault PKI | Your root CA |
| **SPIRE topology** | Single SPIRE Server | Nested SPIRE (CP upstream → Vault CA → cluster SPIRE) | Standalone SPIRE |
| **`api.hexr.cloud` egress** | Required (heartbeat) | Required (heartbeat) | ❌ None |
| **Evidence location** | Customer Postgres | Customer Postgres | Customer Postgres |
| **HIPAA BAA** | ✅ | ✅ (zero cloud CA involvement) | ✅ |
| **FedRAMP IL5 / CMMC L3** | Case-by-case | ✅ | ✅ |
| **Setup time** | ~1 hour | ~2 hours | ~4 hours |
| **License** | Enterprise | Enterprise | Enterprise (air-gap add-on) |

## What Deploys Into Your Cluster

Every mode deploys the same Helm chart into `hexr-system`:

| Component | Purpose |
|---|---|
| **SPIRE Server + Agent** | Per-process SVID issuance |
| **hostPID DaemonSet** | Attests every subprocess on every node |
| **Auto-Registrar** | Watches for pods with `hexr.io/*` labels, creates SPIRE entries |
| **Credential Injector** | JWT-SVID → AWS STS / GCP WIF / Azure exchange |
| **Hexr Vault** | SPIFFE-native secret storage (AES-256-GCM, customer Postgres) |
| **Gateway** | OpenAPI → MCP tool adapter, credential injection |
| **A2A Sidecar** | JSON-RPC 2.0 agent-to-agent protocol with SPIFFE mTLS |
| **Evidence API** | Writes signed evidence rows to customer Postgres |
| **OPA** | Policy enforcement sidecar — fail-closed, per-process |
| **OTel Collector** | Telemetry pipeline → your Prometheus / Jaeger |
| **Envoy** | mTLS sidecar in every agent pod |
