## Deployment Comparison

| | Hexr Cloud | Hybrid | Self-Hosted | Air-Gapped |
|---|---|---|---|---|
| **Infrastructure** | Hexr-managed GKE | Your runtime + Hexr control plane | Your cluster | Your cluster, zero internet |
| **Trust Domain** | `hexr.cloud` | Federated | Customer-owned | Customer-owned |
| **Identity** | Hexr root CA | SPIFFE Federation | Customer SPIRE | Customer SPIRE |
| **Billing** | HCU credits | HCU + infrastructure | License | License |
| **Setup Time** | 5 minutes | 1 hour | 2-4 hours | 2-4 hours |
| **Auth** | SSO / API keys | SSO + SPIFFE Federation | LDAP/AD/SAML/OIDC | LDAP/AD/SAML |
| **Status** | ✅ Available | 🔜 Coming Soon | ✅ Available | ✅ Available |

## Hexr Cloud (Managed SaaS)

Everything managed by Hexr. You write agents, we handle the rest.

<Steps>
  <Step title="Sign up" icon="user-plus">
    Create an account at [hexr.dev](https://hexr.dev) and get an API key.
  </Step>
  <Step title="Authenticate" icon="key">
    ```bash
    hexr login
    ```
  </Step>
  <Step title="Build, push, deploy" icon="rocket">
    ```bash
    hexr build && hexr push --cloud && hexr deploy --cloud
    ```
  </Step>
  <Step title="Monitor" icon="chart-line">
    Dashboard at [app.hexr.cloud](https://app.hexr.cloud) — agents, traces, identity graph, compliance.
  </Step>
</Steps>

**What Hexr Cloud manages for you:**

  <Card title="Cloud API" icon="server">REST API for tenant management, HCU metering, API keys.</Card>
  <Card title="Agent Runtime" icon="robot">GKE Autopilot — auto-scaling, auto-healing, zero ops.</Card>
  <Card title="SPIRE Identity" icon="fingerprint">Hexr root CA. Per-process SVIDs issued automatically.</Card>
  <Card title="Observability" icon="chart-mixed">OTel → Prometheus → Grafana → Jaeger. Pre-configured.</Card>
  <Card title="Cloud SQL" icon="database">PostgreSQL for SPIRE entries, metering, and Vault data.</Card>

</CardGroup>

## Self-Hosted (On-Premises / Private Cloud)

Your infrastructure, your data, your control. Hexr ships as Terraform modules + Helm charts.

<Steps>
  <Step title="Provision infrastructure" icon="cloud">
    ```bash
    terraform init && terraform apply  # VPC, K8s cluster, databases, DNS
    ```
    Works on **AWS EKS, GCP GKE, Azure AKS, bare-metal K8s, DigitalOcean**.
  </Step>
  <Step title="Deploy Hexr runtime" icon="cube">
    ```bash
    helm install hexr-runtime hexr/hexr-runtime -n hexr-system -f values.yaml
    ```
    Deploys SPIRE, Vault, Gateway, Credential Injector, and full observability stack.
  </Step>
  <Step title="Deploy agents" icon="robot">
    ```bash
    hexr build && hexr push && hexr deploy
    ```
    Same CLI, same workflow — agents run in your cluster.
  </Step>
</Steps>

**What deploys into your `hexr-system` namespace:**

<CardGroup cols={3}>
  <Card title="SPIRE Server" icon="fingerprint">Your own certificate authority. Customer-owned trust domain.</Card>
  <Card title="Hexr Vault" icon="lock">SPIFFE-native secrets. AES-256-GCM. Data never leaves your network.</Card>
  <Card title="Hexr Gateway" icon="door-open">OpenAPI → MCP tool adapter. Credential injection from Vault.</Card>
  <Card title="Credential Injector" icon="key">JWT-SVID → STS exchange for AWS, GCP, Azure.</Card>
  <Card title="OTel Stack" icon="chart-mixed">Collector, Prometheus, Jaeger, Grafana — all self-hosted.</Card>
  <Card title="Air-Gapped" icon="shield-halved">Zero outbound connectivity mode. All images pre-loaded.</Card>
</CardGroup>

## Hybrid Cloud (Coming Soon)

Your agents run in your infrastructure. Hexr manages identity, observability, and tooling from the cloud. SPIFFE federation bridges trust domains without exposing credentials.

<CardGroup cols={2}>
  <Card title="Hexr Control Plane (Cloud)" icon="cloud">
    - **Cloud API** — tenant management, config, compliance
    - **Root CA** — Hexr-operated SPIRE Server
    - **Dashboard** — monitoring, identity graph, audit
  </Card>
  <Card title="Your Data Plane (On-Prem)" icon="building">
    - **Customer K8s** — your cluster, your network
    - **Nested SPIRE** — federates with Hexr's root CA
    - **Agent Pods** — compute never leaves your network
  </Card>
</CardGroup>

<Note>
  **SPIFFE Federation** bridges the two trust domains. Your SPIRE server and Hexr's root CA 
  exchange trust bundles — agents in your cluster get Hexr-federated identities without 
  exposing credentials or data. This is an **additive** configuration change, not a redesign.
</Note>

## Architecture Guarantee

Hexr is designed so that **hybrid is additive, not a redesign**. The same Helm charts and agent code work across all models. Moving between deployment models requires changing configuration, not code.

```
Today:
  hexr.cloud trust domain  →  Hexr Cloud SaaS
  acme.example.com         →  Self-Hosted (Terraform + Helm)

Post-funding (additive):
  hexr.cloud ⟷ acme.example.com  →  SPIFFE Federation (config change only)
```
