## Architecture

Every Hexr agent pod runs as a **bare Kubernetes Pod** (not a Deployment) with these containers:

```mermaid
graph TB
    subgraph "Agent Pod"
        INIT["init: install-hexr-sdk"]
        AGENT["Agent Container<br/>(your code)"]
        ENVOY["Envoy Proxy Sidecar"]
        A2A["A2A Sidecar"]
        PID["PID Mapper"]
    end

    subgraph "Cluster Services"
        SPIRE["SPIRE Server + Agent"]
        REG["Auto-Registrar"]
        CRED["Credential Injector"]
        VAULT["Hexr Vault"]
        GW["Hexr Gateway"]
        SAND["Sandbox Engine"]
        GUARD["LLM Guard"]
        VALKEY["Valkey (Redis)"]
        OTEL["OTel Collector"]
        OPA["OPA (sidecar per pod)"]
    end

    AGENT --> ENVOY --> CRED
    AGENT --> A2A
    PID --> SPIRE
    AGENT --> VAULT
    AGENT --> GW
    AGENT --> SAND
    AGENT --> GUARD
    REG --> SPIRE
```

---

## Pod Containers

| Container | Purpose | Port |
|-----------|---------|------|
| [Agent Container](/platform/agent-container) | Your Python code + Hexr SDK | — |
| [Envoy Proxy](/platform/envoy-proxy) | mTLS termination, traffic routing, OPA sidecall | 15001 |
| [A2A Sidecar](/platform/a2a-sidecar) | Agent-to-agent protocol handler | 8090 |
| [PID Mapper](/platform/pid-mapper) | Per-process identity → SPIRE registration | — |
